# Hoverboard FOC 固件 PWM 控制信号逻辑总结

本篇文档总结了 STM32 主固件如何处理典型的 RC 接收机 PWM 标准信号。

## 1. 硬件连接与引脚 (Pinout)
根据 `config.h` 和 `control.c` 的宏定义，PWM 信号通常连接在传感器板接口上：

- **右侧接口 (`CONTROL_PWM_RIGHT`)**:
  - **CH1 (转向/Steer)**: 引脚 `PB10`
  - **CH2 (速度/Speed)**: 引脚 `PB11`
- **左侧接口 (`CONTROL_PWM_LEFT`)**:
  - **CH1 (转向/Steer)**: 引脚 `PA2`
  - **CH2 (速度/Speed)**: 引脚 `PA3`

> [!CAUTION]
> 传感器接口的红色线通常是 15V，**绝对不能**直接接到 RC 接收机或单片机上，否则会烧毁。请仅使用 GND 和信号线。

## 2. 捕获机制 (Capture Mechanism)
固件没有直接使用定时器的输入捕获功能（因为引脚冲突），而是采用了 **GPIO 外部中断 (EXTI) + 定时器计数器** 的方式：

1. **定时器基准**: 使用 `TIM2` 作为基准，计数频率配置为 1MHz (1us/tick)。
2. **双边沿触发**: 引脚配置为 `GPIO_MODE_IT_RISING_FALLING`。
3. **中断回调 (`PWM_ISR_CHx_Callback`)**:
   - **上升沿**: 记录当前的 `TIM2->CNT` 值作为脉冲起始。特别地，如果两个频道都在高电平，可能会重置计时器以防溢出偏移。
   - **下降沿**: 计算当前 `TIM2->CNT` 与起点的差值，得出脉冲宽度（单位：us）。

## 3. 数值转换与映射 (Mapping)
RC 标准信号通常为 1000us (最小) 到 2000us (最大)，1500us 为中立点。

1. **脉宽捕获**:
   - 原始脉冲范围：900us ~ 2100us 被认为是有效信号。
   - 归一化：`pwm_captured_chx_value = CLAMP(脉宽, 1000, 2000) - 1000`。
   - 此时数值范围为 **[0, 1000]**，中立点为 **500**。

2. **输入缩放 (`util.c`)**:
   - 为了适应 FOC 算法的 [-1000, 1000] 内部命令格式，固件执行以下计算：
   - `input_raw = (pwm_captured_chx_value - 500) * 2`
   - **结果**:
     - 1000us -> -1000 (反向行驶 / 全左转)
     - 1500us -> 0 (停止 / 直行)
     - 2000us -> 1000 (正向行驶 / 全右转)

## 4. 安全与超时保护 (Safety)
- **超时监测**: `PWM_SysTick_Callback` 每毫秒递增一次超时计数器。
- **失控保护**: 如果超过 **500ms** 没有收到有效的 PWM 边沿中断，固件会将该频道的原始捕获值重置为 **500 (中立)**。
- **全局超时**: 如果所有输入都丢失，系统会进入 `SAFE_STATE` (命令清零，切断输出)。

## 5. ESP32 到 STM32 的模拟建议
如果您希望使用 ESP32 模拟 PWM 信号给 STM32：
1. 配置 ESP32 的 LEDC 发送 **50Hz** (20ms 周期) 的 PWM 信号。
2. 脉冲宽度在 **1.0ms 到 2.0ms** 之间变化。
3. 确保电平为 **3.3V**。
